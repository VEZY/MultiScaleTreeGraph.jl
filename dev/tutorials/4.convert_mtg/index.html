Converting MTGs

We can do a lot using the MTG format, but sometimes we want our data in another format.

That's why MultiScaleTreeGraph.jl provides functions to expose an MTG as a Tables.jl source, or convert it into a graph.

MTG to table

To get a unified table view of the MTG, you can use:

tbl = to_table(mtg)
MultiScaleTreeGraph.ColumnTable([:node_id, :symbol, :scale, :index, :link, :parent_id, :symbols, :scales, :description, :XEuler, :Length, :Width, :isAlive, :dateDeath], Dict(:description => 9, :symbols => 7, :parent_id => 6, :scale => 3, :node_id => 1, :scales => 8, :XEuler => 10, :dateDeath => 14, :index => 4, :symbol => 2…), AbstractVector[[1, 2, 3, 4, 5, 6, 7], [:Scene, :Individual, :Axis, :Internode, :Leaf, :Internode, :Leaf], [0, 1, 2, 3, 3, 3, 3], [0, 0, 0, 0, 0, 1, 0], [:/, :/, :/, :/, :+, :<, :+], Union{Missing, Int64}[missing, 1, 2, 3, 4, 4, 6], Union{Missing, Vector{String}}[["Scene", "Individual", "Axis", "Internode", "Leaf"], missing, missing, missing, missing, missing, missing], Union{Missing, Vector{Int64}}[[0, 1, 2, 3, 3], missing, missing, missing, missing, missing, missing], Union{Missing, MultiScaleTreeGraph.ColumnTable}[MultiScaleTreeGraph.ColumnTable([:LEFT, :RIGHT, :RELTYPE, :MAX], Dict(:LEFT => 1, :RIGHT => 2, :RELTYPE => 3, :MAX => 4), AbstractVector[["Internode", "Internode"], Vector{SubString{String}}[["Internode", "Leaf"], ["Internode", "Leaf"]], ["+", "<"], ["?", "?"]]), missing, missing, missing, missing, missing, missing], Union{Missing, Float64}[missing, missing, missing, 1.0, missing, 180.0, missing], Union{Missing, Float64}[missing, missing, missing, 0.1, 0.2, 0.1, 0.2], Union{Missing, Float64}[missing, missing, missing, 0.02, 0.1, 0.02, 0.1], Union{Missing, Bool}[missing, missing, missing, missing, 0, 1, 1], Union{Missing, Dates.Date}[missing, missing, missing, missing, Dates.Date("2022-08-24"), missing, missing]])

The unified table reports MTG topology columns plus all attributes:

  • node_id: the unique ID of the node in the whole MTG
  • symbol: the node symbol
  • scale: the node scale
  • index: the node index
  • parent_id: the node's parent id
  • link: the link between the node and its parent

If you only need a few attributes, you can select them directly:

to_table(mtg, vars=[:Width, :Length])
MultiScaleTreeGraph.ColumnTable([:node_id, :symbol, :scale, :index, :link, :parent_id, :Width, :Length], Dict(:symbol => 2, :index => 4, :link => 5, :Width => 7, :Length => 8, :parent_id => 6, :scale => 3, :node_id => 1), AbstractVector[[1, 2, 3, 4, 5, 6, 7], [:Scene, :Individual, :Axis, :Internode, :Leaf, :Internode, :Leaf], [0, 1, 2, 3, 3, 3, 3], [0, 0, 0, 0, 0, 1, 0], [:/, :/, :/, :/, :+, :<, :+], Union{Missing, Int64}[missing, 1, 2, 3, 4, 4, 6], Union{Missing, Float64}[missing, missing, missing, 0.02, 0.1, 0.02, 0.1], Union{Missing, Float64}[missing, missing, missing, 0.1, 0.2, 0.1, 0.2]])

It is also possible to get a per-symbol table:

leaf_tbl = to_table(mtg, symbol=:Leaf)
MultiScaleTreeGraph.ColumnTable([:node_id, :Length, :Width, :dateDeath, :isAlive], Dict(:Length => 2, :Width => 3, :dateDeath => 4, :isAlive => 5, :node_id => 1), AbstractVector[[5, 7], Union{Nothing, Float64}[0.2, 0.2], Union{Nothing, Float64}[0.1, 0.1], Union{Nothing, Dates.Date}[Dates.Date("2022-08-24"), nothing], Union{Nothing, Bool}[0, 1]])

Per-symbol tables can also select attributes:

to_table(mtg, symbol=:Leaf, vars=[:Width, :Length])
MultiScaleTreeGraph.ColumnTable([:node_id, :Width, :Length], Dict(:Length => 3, :Width => 2, :node_id => 1), AbstractVector[[5, 7], Union{Nothing, Float64}[0.1, 0.1], Union{Nothing, Float64}[0.2, 0.2]])

You can also materialize directly into a sink when available in your environment:

using DataFrames
df = to_table(mtg, vars=[:Width, :Length], sink=DataFrame)

If you use DataFrames.jl in your own project, DataFrame(mtg) still works because MTGs implement the Tables.jl interface.

using DataFrames
df = DataFrame(mtg)

MTG to MetaGraph

We can convert an MTG into a MetaGraph using MetaGraph():

mg = MetaGraph(mtg)
Meta graph based on a Graphs.SimpleGraphs.SimpleDiGraph{Int64} with vertex labels of type Int64, vertex metadata of type MultiScaleTreeGraph.ColumnarAttrs, edge metadata of type Symbol, graph metadata given by "MTG", and default weight 1.0

This is particularly useful if you want to benefit from the algorithm provided by Graphs.jl and MetaGraphsNext.jl, such as writing into more standard formats such as DOTFormat or MGFormat (or any other available from GraphIO.jl), plotting with GraphPlot.jl or NetworkLayout.jl, or computing e.g. flow with GraphsFlows.jl.
